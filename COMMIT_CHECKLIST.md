# 🚀 提交前检查清单 (手动版)

> 由于环境配置问题，请手动执行以下检查后再提交代码

## ⚡ 快速检查 (3分钟)

### 1. 代码质量检查
```bash
# 在终端中依次执行以下命令
npm run check      # TypeScript 类型检查
npm run lint       # 代码规范检查
npm run build      # 构建测试
```

**预期结果**: 所有命令都应正常完成，无错误信息

### 2. 关键文件检查
确保以下文件存在：
- [ ] `src/App.tsx` - 主应用组件
- [ ] `src/main.tsx` - 应用入口
- [ ] `src/pages/HomePage.tsx` - 首页组件
- [ ] `src/components/NavigationBar.tsx` - 导航组件

### 3. 基础功能验证
```bash
# 启动开发服务器
npm run dev
```
启动后手动检查：
- [ ] 应用正常启动，显示首页
- [ ] 浏览器控制台无错误信息
- [ ] 导航栏可以正常点击切换页面

## 🔍 详细验证 (可选，建议5分钟)

### UI 组件检查
- [ ] 首页统计数据正常显示
- [ ] 可以访问所有详情页面 (体重、运动、血压、血糖)
- [ ] 移动端模式显示正常

### 核心功能检查
- [ ] 可以添加健康记录
- [ ] 数据刷新后不丢失
- [ ] 图表组件正常渲染

---

## ❌ 阻塞性问题 (发现任何一项都不得提交)

- 应用无法启动
- TypeScript 类型检查失败
- ESLint 检查失败
- 构建过程报错
- 控制台有 JavaScript 错误

## ✅ 提交标准

只有满足以下条件才能提交：
1. 所有自动化检查通过
2. 应用正常启动
3. 核心功能可用
4. 无控制台错误

---

## 📝 提交信息模板

```bash
git add .
git commit -m "feat/fix/docs: 简短描述"
```

**提交类型**:
- `feat`: 新功能
- `fix`: 修复问题
- `docs`: 文档更新
- `refactor`: 代码重构
- `style`: 样式调整
- `test`: 测试相关

---

**记住**: 3分钟的检查可以避免后续数小时的调试时间！ 🎉
