## 🎯 冒烟测试核心原则

### 测试目标
- **快速验证**: 在 5-10 分钟内完成核心功能验证
- **关键路径**: 覆盖用户最常用的核心功能流程
- **构建稳定性**: 确保新构建版本基本可用，无阻塞性问题
- **风险控制**: 提前发现可能导致测试无法继续的严重问题

### 测试时机
✅ **必须执行冒烟测试的场景**:
- 每次代码提交前
- 合并到 `main` 分支前
- 部署到生产环境前
- 依赖包更新后
- 重大功能重构后

## 📋 核心验收检查清单

### 🚀 1. 应用启动与加载 (必须通过)

#### [ ] 1.1 应用启动
- **检查内容**: 应用能够正常启动，无白屏、无控制台错误
- **验证方法**:
  ```bash
  npm run dev  # 开发环境启动
  npm run build && npm run preview  # 生产构建验证
  ```
- **预期结果**:
  - 应用正常加载，显示首页内容
  - 控制台无 JavaScript 错误
  - 网络面板无 404/500 错误

#### [ ] 1.2 资源加载
- **检查内容**: 所有静态资源正确加载
- **验证方法**:
  - 检查 Network 面板
  - 验证 CSS、JS、图片资源状态
- **预期结果**:
  - 所有资源返回 200 状态码
  - 无资源加载失败警告
  - 字体文件正确显示

### 🧭 2. 路由导航 (必须通过)

#### [ ] 2.2 导航组件功能

## 🚨 阻塞性问题清单

**出现以下任何问题，都不得提交或部署：**

### 严重问题 (Must Fix)
1. **应用无法启动** - 白屏、崩溃或控制台报错
2. **路由失效** - 任何页面无法访问
3. **核心功能不可用** - 无法添加/查看健康数据
4. **数据丢失** - localStorage 数据读写异常
5. **构建失败** - `npm run build` 报错
6. **类型错误** - TypeScript 检查失败

### 高优先级问题 (Should Fix)
1. **UI 组件渲染异常** - 组件显示错位或缺失
2. **导航功能异常** - 导航栏不工作
3. **响应式布局失效** - 移动端无法正常使用
4. **性能严重下降** - 页面加载超过 5 秒

## 📝 验收测试流程

### 自动化检查 (每次提交前)
```bash
# 1. 类型检查
npm run check

# 2. 代码规范检查
npm run lint

# 3. 构建测试
npm run build

# 4. 开发服务器启动测试
npm run dev
```

### 手动功能验证 (每次提交前)
1. **基础流程测试 (2分钟)**
   - 启动应用
   - 访问所有路由
   - 测试添加一条记录

2. **UI 检查 (2分钟)**
   - 检查主要页面布局
   - 验证响应式显示
   - 确认组件样式正确

3. **数据功能测试 (3分钟)**
   - 测试数据添加
   - 验证数据持久化
   - 检查数据展示
