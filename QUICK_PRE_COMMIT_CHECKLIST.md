# 快速提交前检查清单 🚀

> 每次提交代码前，请快速确认以下项目 (预计 3-5 分钟)

## ⚡ 自动化检查 (1分钟)

运行以下命令确保基础检查通过：
```bash
npm run smoke-test
```

或者手动执行：
```bash
npm run check      # TypeScript 检查
npm run lint       # 代码规范检查
npm run build      # 构建测试
```

## 🔍 手动快速验证 (2-4分钟)

### ✅ 基础功能检查
- [ ] **应用启动**: `npm run dev` 正常启动，无控制台错误
- [ ] **首页显示**: 首页内容完整显示，布局正常
- [ ] **路由导航**: 点击导航栏可正常跳转到所有页面
- [ ] **数据功能**: 可以成功添加一条健康记录
- [ ] **数据持久化**: 刷新页面后数据不丢失

### ✅ 显示效果检查
- [ ] **移动端**: 在开发者工具移动端模式下显示正常
- [ ] **组件样式**: 所有 UI 组件显示正确，无布局错乱
- [ ] **图表显示**: 包含图表的页面数据正常展示

### ✅ 特殊功能检查 (如已实现)
- [ ] **语音功能**: 语音录制按钮响应正常
- [ ] **数据可视化**: TrendChart 和 DetailChart 正常渲染

---

## 🚫 阻塞性问题 (发现任何一项都不得提交)

- ❌ 应用无法启动或白屏
- ❌ TypeScript 类型检查失败
- ❌ ESLint 规范检查失败
- ❌ 构建过程报错
- ❌ 任何页面无法访问
- ❌ 核心功能不可用
- ❌ 控制台有 JavaScript 错误

## 📝 提交前确认

```
□ npm run smoke-test 通过
□ 应用正常启动
□ 所有路由可访问
□ 核心功能正常
□ 移动端显示正常
□ 无控制台错误
□ 代码已格式化
```

## 🎯 提交信息规范

使用清晰的提交信息格式：
```
feat: 添加体重数据图表功能
fix: 修复移动端导航栏显示问题
docs: 更新项目文档
```

---

## ⚠️ 常见问题解决

### 类型检查失败
```bash
# 查看具体错误
npm run check
# 修复类型错误后重新检查
```

### 代码规范问题
```bash
# 查看具体问题
npm run lint
# 部分问题可自动修复
npx eslint . --fix
```

### 构建失败
```bash
# 清理缓存重新构建
rm -rf node_modules dist
npm install
npm run build
```

### 依赖问题
```bash
# 重新安装依赖
rm -rf node_modules package-lock.json
npm install
```
